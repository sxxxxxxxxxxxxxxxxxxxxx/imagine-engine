# 🧪 AI 图片比例控制 - 详细测试指南

## 🎯 核心改进

### 第二次优化重点

1. **更强的编辑指令** ✅
   - 从 "Generate" 改为 "EDIT/PAINT on canvas"
   - 明确要求保持画布原始尺寸
   - 强调这是"修改画布"而非"生成新图"

2. **优化的空白画布** ✅
   - 使用浅灰色背景（#F5F5F5）而非纯白
   - 添加细微边框，让AI识别画布边界
   - 更容易被AI识别为待编辑的画布

3. **详细的调试日志** ✅
   - 每个关键步骤都有日志输出
   - 生成后自动验证图片尺寸
   - 帮助快速定位问题

4. **消息顺序优化** ✅
   - 图片在前，文字在后
   - AI首先看到画布，然后读取编辑指令

## 🛠️ 调试技巧

### 1. 查看空白画布
在控制台运行：
```javascript
import { createBlankImageByRatio } from '@/utils/imageGenerator';
const blank = createBlankImageByRatio('16:9');
```

### 2. 查看网络请求
1. 开发者工具 → Network 标签
2. 找到 `/api/generate` 请求
3. 点击查看 Request Payload
4. 检查 `baseImage` 字段是否存在
5. 检查是否有两个 content 项（image_url + text）

### 3. 比较文本模式 vs 画布模式
```javascript
// 无参考图 → 应该使用画布模式
```

## ⚙️ 核心机制

### 空白画布特征
```
颜色: #F5F5F5 (浅灰色)
边框: #E0E0E0 (更浅的灰)
边框宽度: 2px
目的: 让AI明确识别这是待编辑的画布
```

### 提示词策略
```
有画布时:
"Edit this blank canvas by painting: [用户描述]"
"MAINTAIN the EXACT dimensions of the canvas"
"Paint on the canvas, don't create new image"

无画布时:
"Generate an image: [用户描述]"
（传统文生图）
```

### 消息顺序
```
content: [
  { type: 'image_url', image_url: { url: blankCanvas } },  // ⬅️ 图片在前
  { type: 'text', text: editInstructions }                 // ⬅️ 文字在后
]
```

## 💡 提示

### 关键改进点
1. **浅灰色画布** - 比纯白更容易被AI识别
2. **边框标记** - 明确画布边界
3. **编辑指令** - "EDIT/PAINT"而非"GENERATE"
4. **图片优先** - 让AI先看到画布
5. **多重强调** - 多次要求保持尺寸

### 下一步
如果还是无法控制比例，可以尝试：
1. 增加画布的视觉特征（网格、参考线等）
2. 调整提示词更加明确
3. 更换不同的 AI 模型测试
4. 联系 API 提供商确认图生图支持
